Introduction to HTML

01
What is HTML?

HTML (HyperText Markup Language) is the standard language for creating web pages. It describes the structure of web content using a system of elements represented by tags. Browsers read HTML and render it as visual pages. HTML is NOT a programming language — it's a markup language.

✦

HTML5 is the current version. It introduced semantic tags, multimedia support, canvas, form validation, data attributes, and much more.

Basic HTML Page 
<!-- This is an HTML comment -->
<!DOCTYPE html>           <!-- Declares HTML5 -->
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to my first web page.</p>
  </body>
</html>
<!DOCTYPE html>
Document type declaration. Tells browser to use HTML5 mode. Always the first line.
<html>
Root element. Wraps entire page. Use lang attribute for language.
<head>
Contains metadata — not visible on page. Holds title, styles, scripts, meta tags.
<body>
All visible content goes here. Everything the user sees on the page.

Document Structure

02
Block vs Inline Elements

Block elements start on a new line and take full width: <div>, <p>, <h1>–<h6>, <section>, etc.
Inline elements flow within text: <span>, <a>, <strong>, <em>, <img>, etc.

Headings, Paragraphs & Containers 
<!-- Headings: h1 is most important, h6 least -->
<h1>Main Heading (page title)</h1>
<h2>Sub Heading</h2>
<h3>Section Heading</h3>
<h4>Sub-section</h4>
<h5>Small heading</h5>
<h6>Smallest heading</h6>

<!-- Paragraphs -->
<p>This is a paragraph of text.</p>

<!-- Line break (self-closing) -->
<p>Line one<br>Line two</p>

<!-- Horizontal rule -->
<hr>

<!-- Generic block container -->
<div class="container">
  <p>Block-level content</p>
</div>

<!-- Generic inline container -->
<p>Text with <span style="color:red">red part</span> inline.</p>

Links & Anchors

04
Anchor Tag — All Types 
<!-- External link (opens in new tab) -->
<a href="https://google.com" target="_blank" rel="noopener noreferrer">
  Visit Google
</a>

<!-- Internal link to another page -->
<a href="about.html">About Us</a>

<!-- Jump to section on same page (anchor) -->
<a href="#contact">Go to Contact section</a>
<section id="contact">...</section>

<!-- Email link -->
<a href="mailto:hello@example.com">Email us</a>

<!-- Phone link -->
<a href="tel:+919876543210">Call us</a>

<!-- Download link -->
<a href="file.pdf" download="my-file.pdf">Download PDF</a>

<!-- Image as link -->
<a href="https://example.com">
  <img src="logo.png" alt="Logo">
</a>
⚠

Always add rel="noopener noreferrer" when using target="_blank" to prevent security vulnerabilities (reverse tabnapping).

Images & Media

05
Image Tags & Attributes 
<!-- Basic image (alt is REQUIRED for accessibility) -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- With dimensions -->
<img src="photo.jpg" alt="Sunset" width="800" height="600">

<!-- Lazy loading (performance!) -->
<img src="photo.jpg" alt="Sunset" loading="lazy">

<!-- Figure with caption -->
<figure>
  <img src="photo.jpg" alt="Mumbai skyline">
  <figcaption>The beautiful Mumbai skyline at night.</figcaption>
</figure>

<!-- Responsive images with srcset -->
<img
  src="photo-800.jpg"
  srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1600.jpg 1600w"
  sizes="(max-width: 600px) 400px, (max-width: 1200px) 800px, 1600px"
  alt="Responsive image"
>

<!-- Picture element (art direction) -->
<picture>
  <source media="(max-width: 600px)" srcset="mobile.jpg">
  <source media="(max-width: 1200px)" srcset="tablet.jpg">
  <img src="desktop.jpg" alt="Responsive art-directed image">
</picture>

Tables

07
Full Table with Accessibility 
<table>
  <caption>Monthly Sales Report</caption>  <!-- table title -->
  
  <thead>                             <!-- header group -->
    <tr>
      <th scope="col">Month</th>
      <th scope="col">Revenue</th>
      <th scope="col">Growth</th>
    </tr>
  </thead>
  
  <tbody>                             <!-- data body -->
    <tr>
      <td>January</td>
      <td>₹1,20,000</td>
      <td>+12%</td>
    </tr>
    <tr>
      <td colspan="2">February (merged cell)</td>  <!-- colspan -->
      <td>+8%</td>
    </tr>
  </tbody>
  
  <tfoot>                             <!-- footer group -->
    <tr>
      <td>Total</td>
      <td>₹2,40,000</td>
      <td>+10%</td>
    </tr>
  </tfoot>
</table>
ℹ

Use colspan to span columns and rowspan to span rows. scope="col" or scope="row" on <th> improves screen reader accessibility.

Semantic HTML5

09
Why Semantic HTML?

Semantic tags tell the browser (and screen readers, search engines) the meaning of content, not just how it looks. Use <article> instead of <div class="article">. This improves SEO, accessibility, and code readability.

Semantic Page Structure 
<body>

  <header>                          <!-- Site header / hero -->
    <nav>                           <!-- Navigation links -->
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
      </ul>
    </nav>
  </header>

  <main>                            <!-- Primary content (only ONE per page) -->

    <section>                      <!-- Thematic grouping with heading -->
      <h2>Latest Articles</h2>

      <article>                    <!-- Self-contained content (blog post) -->
        <header>
          <h3>Article Title</h3>
          <time datetime="2025-01-15">January 15, 2025</time>
        </header>
        <p>Article content...</p>
        <footer>
          <address>By <a href="mailto:a@b.com">Author Name</a></address>
        </footer>
      </article>

    </section>

    <aside>                         <!-- Sidebar / tangentially related -->
      <h3>Related Links</h3>
    </aside>

    <details>                      <!-- Expandable/collapsible content -->
      <summary>Click to expand</summary>
      <p>Hidden content revealed on click!</p>
    </details>

  </main>

  <footer>                          <!-- Site footer -->
    <p>&copy; 2025 My Website</p>
  </footer>

</body>

Meta & Head Tags

10
Complete Head Section 
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">

  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- SEO meta tags -->
  <title>Page Title | Brand Name</title>
  <meta name="description" content="160 char description for search engines">
  <meta name="keywords" content="html, css, web development">
  <meta name="author" content="Your Name">
  <meta name="robots" content="index, follow">

  <!-- Open Graph (social media sharing) -->
  <meta property="og:title" content="Page Title">
  <meta property="og:description" content="Description for Facebook/LinkedIn">
  <meta property="og:image" content="https://example.com/og-image.jpg">
  <meta property="og:url" content="https://example.com/page">
  <meta property="og:type" content="website">

  <!-- Twitter card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Page Title">

  <!-- Favicon -->
  <link rel="icon" href="/favicon.ico">
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">

  <!-- CSS -->
  <link rel="stylesheet" href="styles.css">

  <!-- Canonical URL (avoids duplicate content) -->
  <link rel="canonical" href="https://example.com/page">

  <!-- Theme color (mobile browser bar) -->
  <meta name="theme-color" content="#7c6af7">

  <!-- Google Fonts -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">

  <!-- Script (defer = run after HTML parsed) -->
  <script src="app.js" defer></script>
</head>

Canvas & SVG

12
Canvas API 
<!-- Canvas element -->
<canvas id="myCanvas" width="400" height="300">
  Fallback text for non-canvas browsers
</canvas>

<script>
  const canvas = document.getElementById('myCanvas');
  const ctx = canvas.getContext('2d');

  // Draw a filled rectangle
  ctx.fillStyle = '#7c6af7';
  ctx.fillRect(50, 50, 150, 100);

  // Draw a circle
  ctx.beginPath();
  ctx.arc(250, 100, 50, 0, Math.PI * 2);
  ctx.fillStyle = '#3ecfb2';
  ctx.fill();

  // Draw text
  ctx.font = '24px sans-serif';
  ctx.fillStyle = 'white';
  ctx.fillText('Hello Canvas!', 50, 200);
</script>
Inline SVG 
<!-- SVG directly in HTML -->
<svg width="200" height="200" viewBox="0 0 200 200" xmlns="http://www.w3.org/2000/svg">

  <!-- Circle -->
  <circle cx="100" cy="100" r="80" fill="#7c6af7"/>

  <!-- Rectangle -->
  <rect x="20" y="20" width="60" height="60" fill="#3ecfb2" rx="8"/>

  <!-- Line -->
  <line x1="0" y1="0" x2="200" y2="200" stroke="white" stroke-width="2"/>

  <!-- Path (custom shape) -->
  <path d="M 10 80 Q 95 10 180 80" fill="none" stroke="#f79c6a" stroke-width="3"/>

  <!-- Text -->
  <text x="100" y="105" text-anchor="middle" fill="white">SVG</text>

</svg>

HTML APIs & Features

13
Data Attributes, Templates & Custom 
<!-- DATA ATTRIBUTES: store custom data on elements -->
<div
  data-user-id="42"
  data-role="admin"
  data-active="true"
>User Card</div>

<script>
  const el = document.querySelector('div');
  console.log(el.dataset.userId);   // "42"
  console.log(el.dataset.role);     // "admin"
</script>

<!-- HTML TEMPLATE: reusable content (not rendered) -->
<template id="card-template">
  <div class="card">
    <h3 class="card-title"></h3>
    <p class="card-body"></p>
  </div>
</template>

<script>
  const tmpl = document.getElementById('card-template');
  const clone = tmpl.content.cloneNode(true);
  clone.querySelector('.card-title').textContent = 'Title';
  document.body.appendChild(clone);
</script>

<!-- DIALOG (native modal) -->
<dialog id="myDialog">
  <p>Native HTML modal dialog!</p>
  <button onclick="document.getElementById('myDialog').close()">Close</button>
</dialog>
<button onclick="document.getElementById('myDialog').showModal()">Open Modal</button>

<progress value="70" max="100">70%</progress>

<meter value="0.6" min="0" max="1" low="0.25" high="0.75">60%</meter>

Accessibility (A11y)

14
Why Accessibility?

About 15% of the world's population lives with some form of disability. Accessible HTML ensures your website works for everyone — screen reader users, keyboard-only users, and people with visual or motor impairments. It also improves SEO.

ARIA & Accessibility Patterns 
<!-- ARIA ROLES -->
<div role="navigation" aria-label="Main navigation">...</div>
<div role="alert">Form submitted successfully!</div>
<div role="status" aria-live="polite">Loading...</div>

<!-- ARIA LABELS -->
<button aria-label="Close dialog">✕</button>
<input aria-labelledby="email-label">
<span id="email-label">Email Address</span>

<!-- ARIA STATES -->
<button aria-expanded="false" aria-controls="menu">Menu</button>
<nav id="menu" aria-hidden="true">...</nav>

<input aria-required="true" aria-invalid="false">
<div aria-disabled="true"></div>

<!-- SKIP LINKS -->
<a href="#main-content" class="skip-link">Skip to main content</a>

<!-- KEYBOARD NAVIGATION -->
<div tabindex="0">Focusable div</div>    <!-- In tab order -->
<div tabindex="-1">Programmatic focus</div><!-- focus() only -->

<!-- LANG ATTRIBUTE (screen readers) -->
<html lang="en">
<span lang="fr">Bonjour</span>

<!-- VISUALLY HIDDEN (screen reader only) -->
<span style="position:absolute;width:1px;height:1px;overflow:hidden;clip:rect(0,0,0,0)">
  Screen reader only text
</span>
✦

Golden rule: First rule of ARIA is — don't use ARIA. Use native semantic HTML elements first. Only add ARIA when native HTML can't express the needed semantics.
